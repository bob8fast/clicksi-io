üìã Overview

Dynamic pages in Clicksi allow you to create content-driven pages that are stored in the database and rendered dynamically. This system supports multilingual content, custom styling, and flexible layouts.

üéØ Key Features:
  • Database-driven content management
  • Multilingual support (English, Ukrainian, Polish)
  • Custom CSS styling per page
  • Conditional header/footer display
  • Language fallback to English
  • SEO-friendly metadata

üóÑÔ∏è Database Schema

Dynamic pages are stored in the pages table with the following structure:

Field Type Description Required
id string Unique identifier (UUID) Yes
slug string URL slug (e.g., "demo", "about-us") Yes
title string Page title for SEO and display Yes
content JSONContent Main page content (HTML, JSON, or text) Yes
style JSONContent Custom CSS styles for the page No
description string Meta description for SEO No
keywords array SEO keywords No
status string "published", "draft", "archived" Yes
lang string "en", "ua", "pl" Yes
show_title boolean Display page title Yes
show_description boolean Display page description Yes
show_metadata boolean Display last updated date Yes
show_header boolean Include site header Yes
show_footer boolean Include site footer Yes
show_button boolean Show custom button/navigation No
button JSONContent Custom button HTML No

üìù Content Types

The content field supports multiple formats through the JSONContent type:

1. HTML Content

{
  "type": "html",
  "value": "<h1>Welcome to Clicksi</h1><p>Ukrainian creator platform</p>"
}

2. Plain Text

{
  "type": "text",
  "value": "Simple text content that will be escaped for safety"
}

3. JSON Tree Structure

{
  "type": "json",
  "value": [
    {
      "type": "element",
      "tag": "h1",
      "attributes": {"class": "hero-title"},
      "children": [
        {"type": "text", "text": "Dynamic Content"}
      ]
    },
    {
      "type": "element",
      "tag": "p",
      "children": [
        {"type": "text", "text": "Built with JSON structure"}
      ]
    }
  ]
}

4. Direct String

"<h1>Direct HTML string</h1><p>No wrapper object needed</p>"

üåç Language Support

Clicksi supports three languages with automatic fallback functionality:

Supported Languages

  • English (en) - Default language
  • Ukrainian (ua) - Primary target market
  • Polish (pl) - Secondary market

Language Fallback System

When a page is requested in a specific language but doesn't exist, the system automatically falls back to English:

// User requests: /ua/demo
// 1. Try to find page with slug="demo" and lang="ua"
// 2. If not found, fallback to slug="demo" and lang="en"
// 3. If still not found, show 404
‚úÖ Best Practice: Always create an English version of your pages as the fallback. This ensures content is always available regardless of the user's language preference.

üî® Creating Dynamic Pages

Database Insert Example

INSERT INTO pages (
  id, slug, title, content, style, description,
  keywords, status, lang, show_title, show_description,
  show_metadata, show_header, show_footer,
  created_at, updated_at
) VALUES (
  gen_random_uuid(),
  'demo-page',
  'Clicksi Demo Page',
  '{"type": "html", "value": "<h1>Demo Content</h1><p>This is a demo page</p>"}',
  '{"type": "html", "value": ".custom-style { color: #D78E59; }"}',
  'Experience the Clicksi platform demo',
  '["demo", "clicksi", "creators", "brands"]',
  'published',
  'en',
  true,
  true,
  true,
  true,
  true,
  NOW(),
  NOW()
);

Using the API

// Create a new dynamic page
const pageData = {
  slug: 'new-feature',
  title: 'New Feature Announcement',
  content: {
    type: 'html',
    value: '<h1>Exciting New Feature!</h1><p>Details about the feature...</p>'
  },
  description: 'Learn about our latest platform update',
  status: 'published',
  lang: 'en',
  show_title: true,
  show_description: true,
  show_metadata: true,
  show_header: true,
  show_footer: true
};

// POST to your API endpoint
await fetch('/api/pages', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify(pageData)
});

üé® Custom Styling

Each page can have custom CSS styles that are injected into the page head:

Style Format

{
  "type": "html",
  "value": ".hero-section { background: linear-gradient(135deg, #0057b7, #D78E59); padding: 4rem 2rem; } .custom-button { background: #D78E59; color: #171717; }"
}

Best Practices

  • Use CSS custom properties for consistency with theme
  • Scope styles to avoid conflicts: .page-demo .custom-style
  • Test styles across different screen sizes
  • Avoid !important unless necessary
‚ö†Ô∏è Security Note: Custom styles are injected directly into the page. Ensure styles are sanitized and come from trusted sources only.

üõ£Ô∏è URL Routing

Dynamic pages use the following URL structure:

https://clicksi.io/[locale]/[slug]

Examples:
- https://clicksi.io/en/demo
- https://clicksi.io/ua/about-us
- https://clicksi.io/pl/privacy-policy

Route Resolution

  1. Extract locale and slug from URL
  2. Query database for matching page
  3. If not found, try fallback language (English)
  4. If still not found, check for static legal pages
  5. Finally, show 404 if no match found

Static vs Dynamic Priority

The system prioritizes pages in this order:

  1. Database pages - Dynamic content from database
  2. Static legal pages - Privacy policy, terms, etc.
  3. 404 page - Page not found

‚úÖ Best Practices

üîê Security

  • Sanitize all HTML content before storing
  • Validate JSON structure for content field
  • Escape user-generated content
  • Use parameterized queries for database operations

üöÄ Performance

  • Cache frequently accessed pages
  • Optimize images and media assets
  • Minimize custom CSS size
  • Use database indexes on slug and lang fields

üåç Internationalization

  • Always provide English fallback content
  • Use appropriate language codes (en, ua, pl)
  • Consider cultural context in content
  • Test right-to-left languages if needed

‚ôø Accessibility

  • Use semantic HTML structure
  • Include alt text for images
  • Ensure sufficient color contrast
  • Test with screen readers
